# 一、先明确：发明专利的“硬性门槛”

核心是三条（缺一不可）：

### 1️⃣ 新颖性（Novelty）

> 是否“之前没有被公开过”

判断方法（实操）：

- 用关键词 + 同义词 + 英文术语
- 去这些库检索：
  - Google Patents
  - 国家知识产权局
  - WIPO

👉 如果**高度相似方案已经存在 → 基本废掉**
👉 如果只是“方向类似，但机制不同 → 还能做”

------

### 2️⃣ 创造性（Inventive Step）

> 不是“显而易见的改进”

这是**最难的一点**，审查员最爱卡这里。

判断标准（简化）：

- ❌ 把A+B拼一起（常规组合） → 不行
- ❌ 参数调优 / 换模型 → 不行
- ❌ “用AI优化XX”但没新机制 → 很危险
- ✅ 引入**新的机制 / 流程 / 结构**
- ✅ 解决了**现有技术明确存在的问题**
- ✅ 技术效果**显著提升（不是一点点）**

👉 一句话判断：

> “如果我是同行，看完会不会觉得：这不就是顺手就能想到？”

------

### 3️⃣ 实用性（Utility）

> 能不能“落地”，不是纯理论

- ❌ 纯算法证明 / 数学模型
- ❌ 没有应用场景
- ✅ 有明确系统架构
- ✅ 有业务流程（比如你做的审计）
- ✅ 可以工程实现

# 二、你可以用的“选题筛选五步法”（非常关键）

拿你一个选题，按这个流程走👇

------

## Step 1：一句话描述技术方案

必须是这种结构：

> “在【场景】下，通过【技术手段】，解决【具体问题】”

例如（好的）：

- 在审计多文档场景下，通过多Agent协同与结构化RAG，解决事实溯源与一致性问题

👉 如果你说不清 → 这个题目还不成熟

------

## Step 2：找“现有技术最接近的方案”（Closest Prior Art）

这是专利思维核心（论文不强调这个）

你要找到：

- 最像你方案的已有专利 / 论文

然后问：

👉 我比它**多做了什么？**

------

## Step 3：提炼“区别特征”（Differentiating Features）

写成列表：

- Feature A：xxx
- Feature B：xxx
- Feature C：xxx

👉 注意：
这些必须是**结构/流程级创新**，不是效果描述

------

## Step 4：证明“技术效果”

必须是这种逻辑：

> 区别特征 → 导致 → 技术效果

例如：

- 多Agent分工 → 降低信息冲突 → 提高一致性
- 结构化索引 → 降低检索噪声 → 提高准确率

👉 没有这个链条 = 创造性站不住

------

## Step 5：判断是否“值得申请”

用这个标准：

| 维度     | 判断标准             |
| -------- | -------------------- |
| 新颖性   | 是否有高度重复       |
| 创造性   | 是否有机制级创新     |
| 技术深度 | 是否不是“prompt工程” |
| 可保护性 | 是否能写成系统/方法  |
| 商业潜力 | 是否可产品化         |

👉 满足 3.5 条以上 → 可以做

**权利要求书**

**权利要求1（独立权利要求 —— 方法）**  
一种基于审计证据图路径搜索与全局最优节点重排序的法规检索增强生成方法，其特征在于，包括以下步骤：  
（1）将审计查询分解为多个子问题集合；  
（2）针对每个子问题沿不同语义路径并行检索审计法规长文档，构建审计证据图G（节点为文本chunk块或语义聚类中心，边为路径依赖关系与共享节点关系）；  
（3）在所述审计证据图上进行路径搜索，计算每条候选路径的路径一致性评分，并引入跨路径节点冲突约束（当两条或多条路径共享同一节点但语义冲突时，施加冲突惩罚）；  
（4）将候选路径选择转化为带约束的全局优化问题，定义全局优化目标：  
\[ \max_{P} \sum_{p \in P} \text{PathConsistency}(p) - \mu \cdot \text{ConflictPenalty}(p) \]  
其中 \(\mu\) 为冲突惩罚系数；  
（5）**所述全局优化目标由计算设备执行**，通过节点级重排序（Node re-ranking）筛选全局最优路径集合，并融合最优路径输出检索增强生成结果，从而避免传统独立重排序导致的证据不一致问题，实现审计法规长文档检索的精确性与一致性提升。

**权利要求2（独立权利要求 —— 系统）**  
一种基于审计证据图路径搜索与全局最优节点重排序的法规检索增强生成系统，其特征在于，包括查询分解模块、审计证据图构建模块、路径搜索与冲突约束模块、全局优化计算模块、节点重排序模块，以及生成融合模块，所述模块协同实现权利要求1所述方法。

**权利要求3**

**权利要求4**
