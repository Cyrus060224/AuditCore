# AuditCore 注释规范

## 规则（生成任何代码时都必须遵守）

1. **极简原则**
   - 只注释业务逻辑和数据流转意图
   - 不翻译语法本身

2. **禁止 C++ 类比**
   - 不使用任何 C/C++ 概念类比

3. **模块级文档 (Docstrings)**
   - 每个核心类（Class）和函数（Def）必须用三引号写清楚
   - 说明：输入要求、输出内容、可能抛出的异常

4. **跨平台标记**
   - 为解决 Windows/macOS 兼容性而写的代码，必须用 [Cross-Platform] 前缀标注

5. **代码纯净度**
   - 所有 UI 展示文本统一使用英文
   - 变量名、函数名、路径严禁出现中文字符
   - 注释使用中文
